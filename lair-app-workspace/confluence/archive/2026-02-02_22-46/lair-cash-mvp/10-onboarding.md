## 1. 개요

### 목적

* 로그인 없이 퀴즈를 통해 서비스 가치를 먼저 체험
* 10단계 퀴즈 완주 후 가입 유도 (지연된 가입)
* 금 보상으로 가입 동기 부여

### 핵심 원칙

* **무관문 진입** — 로그인 없이 바로 퀴즈 시작
* **쉬운 난이도** — 누구나 완주할 수 있도록 설계
* **보상 체감** — 퀴즈 완주 시 금 10mg 획득
* **스킵 옵션** — 귀찮은 유저를 위한 스킵 기능 (보상 없음)

---

## 2. 사용자 여정

```
1. 서비스 소개 화면
   → 앱 첫 진입 시 서비스 핵심 가치 전달

2. 10단계 퀴즈
   → 로그인 없이 바로 시작
   → 쉬운 금융 퀴즈 10개를 풀며 학습
   → 정답 시 금 획득 축하 팝업 + 애니메이션
   → 스킵 가능 (보상 없이 바로 로그인으로 이동)

3. 완주 축하 화면
   → "금 10mg 발견!" 보상 인지
   → (스킵한 경우 이 화면 건너뜀)

4. 소셜 로그인
   → 이메일 / 구글 / 애플 로그인 선택
   → 금을 저장하기 위해 가입 유도

5. 레퍼럴 코드 입력
   → 레퍼럴 링크로 진입 시 자동 입력 후 스킵
   → 직접 진입 시 코드 입력 화면 표시

6. 홈 화면 이동
   → 가입 완료 후 홈 화면으로 이동
   → 퀴즈 보상 금 10mg 지급 완료 확인 (완주한 경우만)
```

## 3. 화면별 상세

### 3.1 서비스 소개 화면

| 요소 | 내용 |
| --- | --- |
| 헤드라인 | "달러 넣으면 금이 쌓인다" |
| 서브텍스트 | "퀴즈 풀고 금 10mg 받기" |
| CTA | "금 캐러 가기" |

### 3.2 퀴즈 화면

| 요소 | 스펙 |
| --- | --- |
| 진행바 | 1/10 ~ 10/10 표시 |
| 질문 | 1줄 헤드라인 |
| 보기 | 2~4개 선택지 |
| 정답 피드백 | 축하 팝업 + 금 떨어지는 애니메이션 |
| 오답 피드백 | 힌트 제공 → 즉시 재도전 |
| 건너뛰기 | 우측 상단 텍스트 버튼 |

### 3.3 건너뛰기 확인 팝업

```

│  없어요                  │
│                         │
│  [계속하기]   [건너뛰기]   │
└─────────────────────────┘
```

## 6. 기존 유저 처리

| 상황 | 동작 |
| --- | --- |
| 이미 가입된 유저 | 앱 진입 → 바로 홈 화면 |
| 퀴즈 중간 이탈 후 재진입 | 처음부터 다시 시작 (진행 저장 안 함) |
| 퀴즈 스킵 후 가입 안 하고 이탈 | 재진입 시 퀴즈 다시 진행 |
| 가입 완료한 유저가 퀴즈 재진행 원함 | 설정 > 튜토리얼에서 재진행 (보상 없음) |

## 7. 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
| --- | --- | --- |
| 퀴즈 시작률 | 80%+ | 서비스 소개 → 퀴즈 시작 |
| 퀴즈 완주율 | 70%+ | 퀴즈 시작 → 10단계 완료 |
| 퀴즈 스킵률 | <20% | 퀴즈 시작 → 스킵 |
| 가입 전환율 | 80%+ | 퀴즈 완주 또는 스킵 → 로그인 완료 |
| 온보딩 완료율 | 45%+ | 앱 진입 → 홈 화면 도달 |
