# Rule: SDLC 클린 코드 원칙

## 적용 범위
코드를 생성하는 모든 SDLC 에이전트

## 규칙

### 1. 네이밍
- 변수/함수명은 의미를 명확히 전달
- 약어 사용 최소화
- 일관된 케이스 사용 (camelCase, PascalCase 등)

### 2. 함수
- 단일 책임 원칙 (하나의 함수 = 하나의 역할)
- 함수 길이 30줄 이내 권장
- 순수 함수 선호 (side-effect 최소화)

### 3. 구조
- 관심사 분리 (UI, Logic, Data)
- 중복 코드 제거 (DRY)
- 명확한 폴더/파일 구조

### 4. 주석
- 코드로 설명 가능한 부분은 주석 생략
- "왜(Why)"에 대한 주석 작성
- TODO/FIXME 태그 활용

### 5. 에러 처리
- 예외 상황 명시적 처리
- 사용자 친화적 에러 메시지
