# Rule: 테스팅 규칙

## 적용 범위
모든 개발 관련 에이전트

---

## 테스트 커버리지

- **목표**: 80% 이상
- **Unit Tests**: 함수, 유틸리티, 컴포넌트
- **Integration Tests**: API, DB 연동
- **E2E Tests**: 핵심 사용자 플로우 (Playwright)

---

## TDD 워크플로우

1. **RED**: 테스트 먼저 작성
2. **실패 확인**: 테스트 실행 → 실패해야 함
3. **GREEN**: 최소한의 구현
4. **통과 확인**: 테스트 실행 → 통과해야 함
5. **REFACTOR**: 리팩토링
6. **커버리지 확인**: 80% 이상인지 검증

---

## 테스트 원칙

- 테스트는 **독립적**이어야 함
- Mock은 **최소한**으로 사용
- 구현이 아닌 **동작**을 테스트
- 테스트 실패 시 **구현 수정** (테스트가 맞다면)
