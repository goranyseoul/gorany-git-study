# Rule: 요구사항 문서 구조

## 적용 범위
`requirements-planner` 에이전트 및 요구사항 관련 작업

## 필수 문서 구조

### 1. 메인 문서: `docs/1-요구사항/required.md`
**필수 섹션:**
- 📋 문제 (Problem)
- 💡 해결방법 (Solution)
- 🎯 목표 (Goal)
- 📱 **지원 플랫폼** (웹/모바일/PC 표)
- 🔄 사용자 플로우
- ✅ 요구사항 체크리스트

### 2. 세부 문서: `docs/1-요구사항/required_{번호}.md`

**필수 섹션:**

| 섹션 | 필수 | 내용 |
|------|------|------|
| 개요 (ID, **플랫폼**) | ✅ | 지원 플랫폼 명시 |
| 고객의 목소리 | ✅ | 실제 고객 니즈 인용문 |
| 사용자 스토리 | ✅ | AS A / I WANT TO / SO THAT |
| 인수 조건 | ✅ | 고객 관점 체크리스트 |
| 연관 요구사항 | 권장 | 의존성 표시 |

### 3. 사용자 플로우: `docs/요구사항_userflow.html`
Mermaid.js 기반 HTML 다이어그램

---

## + 항목 추가 규칙

사용자가 `required.md`에 `+`로 시작하는 항목 추가 시:
1. 새 요구사항으로 인식
2. 다음 번호 `required_N.md` 파일 생성
3. 체크리스트에 정식 항목으로 추가
4. `+` 제거

- ❌ 기술 스펙: "Google OAuth 연동 구현"
- ✅ 고객 니즈: "간편하게 로그인하고 싶다"
