# Rule: UI/UX 디자인 원칙

## 적용 범위
디자인 관련 모든 에이전트 (concept-designer, detail-designer 등)

---

## 핵심 원칙

### 1. Nielsen의 10가지 휴리스틱
- **가시성**: 시스템 상태를 항상 사용자에게 알려줌
- **일관성**: 플랫폼 관례를 따르고 일관된 패턴 사용
- **에러 방지**: 에러 메시지보다 예방이 우선
- **인식 > 기억**: 사용자가 기억하지 않아도 되게
- **유연성**: 초보자와 전문가 모두 고려
- **미니멀**: 불필요한 정보 제거
- **에러 복구**: 에러 발생 시 복구 방법 제공
- **도움말**: 필요시 쉽게 접근 가능

### 2. Atomic Design 구조
- **Atoms**: 버튼, 입력, 라벨 등 기본 요소
- **Molecules**: Atom 조합 (검색창 = 입력 + 버튼)
- **Organisms**: Molecule 조합 (헤더 = 로고 + 네비 + 검색)
- **Templates**: 페이지 레이아웃
- **Pages**: 실제 콘텐츠

### 3. Gestalt 원칙
- **근접성**: 가까운 요소는 그룹으로 인식
- **유사성**: 비슷한 요소는 관련으로 인식
- **연속성**: 선/곡선을 따라 시선 이동
- **폐쇄성**: 불완전해도 완전한 형태로 인식

## 필수 체크리스트

### 접근성 (a11y)
- [ ] 색상 대비 4.5:1 이상
- [ ] 키보드 네비게이션 가능
- [ ] 포커스 상태 명확
- [ ] alt 텍스트 제공

### 반응형
- [ ] Mobile: 375px
- [ ] Tablet: 768px
- [ ] Desktop: 1024px+

### 인터랙션
- [ ] 클릭 요소에 cursor: pointer
- [ ] 호버/포커스 상태 제공
- [ ] 트랜지션 150-300ms
