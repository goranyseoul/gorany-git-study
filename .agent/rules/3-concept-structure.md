# Rule: 디자인 컨셉 구조

## 적용 범위
`concept-designer` 에이전트 및 디자인 관련 작업

---

## 필수 참조 문서

작업 시작 전 반드시 읽어야 할 문서:
1. `docs/1-요구사항/required.md` 및 세부 요구사항
2. `docs/2-기능정의/features.csv`

---

## 프로세스

### 기본 모드 (로컬)
1. **[필수]** 요구사항 문서 분석
2. **[필수]** 기능정의 분석
3. 화면 목록 및 구성요소 정의 (screens.md)
4. 디자인 시스템 생성 (MASTER.md + 디자인_master.html)
5. 컨셉 설명 문서 작성 (concept.md)

### Stitch 연동 모드 (사용자 요청 시)
- "Stitch로 만들어줘" 요청 시 활성화
- Stitch 웹앱에서 UI 생성 후 API로 가져오기

---

## 출력 구조

### 1. 컨셉 문서: `docs/3-디자인/concept.md`

| 섹션 | 필수 | 내용 |
|------|------|------|
| Stitch 프로젝트 정보 | 선택 | 프로젝트 ID, 링크 |
| 화면 목록 | ✅ | 필요한 화면 체크리스트 |
| 디자인 방향 | ✅ | 컬러, 타이포, 스타일 |

### 2. 화면 설계서: `docs/3-디자인/screens.md`

| 섹션 | 필수 | 내용 |
|------|------|------|
| 화면별 구성요소 | ✅ | 각 화면에 들어갈 UI 요소 |
| 체크리스트 | ✅ | 프로토타입 검수용 |

### 3. 디자인 시스템 요약: `docs/3-디자인/MASTER.md`

| 섹션 | 필수 | 내용 |
|------|------|------|
| 컬러 팔레트 | ✅ | CSS 변수 포함 |
| 타이포그래피 | ✅ | 폰트, 크기 |
| 컴포넌트 스펙 | ✅ | 버튼, 카드, 입력 등 |

### 4. 디자인 시스템 미리보기: `docs/디자인_master.html`

실제 스타일을 브라우저에서 확인할 수 있는 HTML 파일

---

## 금지 사항
- 요구사항/기능정의 없이 디자인 시작 금지
- 임의 화면 추가 금지 (기능에서 도출)
