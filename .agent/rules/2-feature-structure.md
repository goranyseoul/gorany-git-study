# Rule: 기능 정의서 구조

## 적용 범위
`feature-planner` 에이전트 및 기능 정의 관련 작업

## 필수 문서 구조

### 1. 데이터 파일: `docs/2-기능정의/features.csv`

**필수 컬럼:**

| 컬럼 | 필수 | 내용 |
|------|------|------|
| id | ✅ | FT-001 형식 |
| name | ✅ | 기능명 |
| priority | ✅ | MVP / 중요 / 부가 |
| version | ✅ | v1.0 / v1.1 / v2.0 |
| status | ✅ | todo / in-progress / done |

### 2. 시각화: `docs/기능정의_board.html`

---

## 우선순위 분류 기준

| 분류 | 의미 | 기준 |
|------|------|------|
| 🔴 MVP | 없으면 서비스 불가 | 핵심 플로우 필수 |
| 🟡 중요 | 경쟁력 UP | 만족도/편의성 향상 |
| 🟢 부가 | Nice-to-have | 차별화, 수익화 |

---

## 버전 배치

- v1.0 (MVP): 최소 출시 가능 제품
- v1.1: 핵심 개선 및 중요 기능
- v2.0: 차별화 및 확장 기능

## 📋 Notion Epic 생성 규칙

Epic 페이지 생성 시 **반드시 상세 내용도 함께 추가**합니다.

```
## 개요
(기능 설명 1~2줄)

## 인수 조건
- (완료 기준 1)
- (완료 기준 2)
- (완료 기준 3)

## 관련 화면
(연관된 프로토타입 파일명)
```

### API 호출 방식
`post-page` 호출 시 `children` 파라미터에 블록 배열을 포함:
- heading_2: 개요, 인수 조건, 관련 화면
- paragraph: 설명 텍스트
- bulleted_list_item: 인수 조건 항목
