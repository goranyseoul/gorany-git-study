## 1. 핵심 요약 (Executive Summary)

오늘 회의는 ERC8045 표준을 중심으로 AI 에이전트가 블록체인에서 어떻게 독립적인 경제 활동을 수행할 수 있는지 논의했습니다. NotebookLM을 활용한 리서치 결과를 공유하였고, A16Z에서 제안하는 AI 에이전트 + 지갑 결합 모델을 검토했습니다. 또한 Lair 프로젝트 기획서 PR 리뷰, 프로토타입 개발 진행상황, 그리고 Human Protocol 체험을 통해 AI가 사람을 고용하는 실제 사례를 확인했습니다.

## 2. 주요 의사결정 (Key Decisions)

- **ERC8045 표준 학습**: AI 에이전트 + 블록체인 결합에 영감을 얻기 위해 ERC8045 표준을 팀 전체가 학습하기로 결정

- **Human Protocol 탐색**: AI가 돈을 주고 사람을 고용하는 플랫폼 체험

## 4. 상세 내용 (Detailed Notes)

### 4.1. ERC8045 - AI 에이전트 표준

- **NotebookLM 활용**: 긴 문서를 요약하고 Q&A 기반으로 정보를 얻는 데 유용
- **ERC8045 핵심 개념**:
  - AI 에이전트에게 NFT 기반 신원 등록
  - AI 에이전트에게 지갑 부여
  - 평판 레지스트리: 사용자가 AI 에이전트 서비스 후 평점 남김
  - 검증 시스템: 영지식 증명, 스테이킹 등

### 4.2. A16Z - AI 에이전트 경제

- AI 에이전트는 기존 금융 시스템(신용카드, 은행 계좌) 이용 불가 → 블록체인 지갑이 해결책
- 스마트 컨트랙트 + AI = 더 자율적인 작업 수행 가능
- 사례: NFT 아트워크 판매 에이전트, 무담보 차익거래 봇, 자금 모아 영화 제작

### 4.3. 팀 vs 개인 프로젝트

- AI 자동화가 고도화되면 1인이 전체 서비스 개발 가능
- **팀만의 가치**: 개인이 못하는 영역에서 차별화 필요
- 아직 구체적 답은 없음, 지속 논의 필요

### 4.4. Lair 기획서 PR

- **문제**: PR이 한종 개인 브랜치로 잘못 올라감, 폴더 이름이 "lair-cache-module"로 부적절
- **해결**: 폴더 이름을 "기획" 등으로 변경, master에 재제출
- **구조**:
  - 13개 기능 모듈
  - 6개 참조문서
  - 개별 기능별 PRD, 와이어프레임, 테스트스펙, 이벤트 정의
  - 검증 리포트 포함
  - 유저플로우 기반 통합 가이드

### 4.5. 프로토타입 개발 진행

- 금캐러 게임 프로토타입 개발 중
- 기획서 기반으로 ~90% 수준 자동 생성
- 디자인 시스템 적용으로 일관된 UI 구현

### 4.6. 기타 공유

- **비탈릭 - L2 비판**: L2가 단순히 가스비 절감만 제공한다면 의미 없음, 고유한 특장점 필요
- **AI 코딩 팁**: 여러 세션 동시 실행, 별도 동료로 운영하는 모델 추천
- **Human Protocol 체험**: AI가 돈을 주고 사람을 고용하는 플랫폼, 팀원들이 직접 지원 테스트
