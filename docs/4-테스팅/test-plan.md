# 테스트 계획

## 개요

**프로젝트**: 냉장고 재고 관리 앱
**테스트 기간**: 개발 완료 후
**테스트 범위**: Frontend (React Native) + Backend (NestJS)

---

## 테스트 범위

| 영역 | 범위 | 우선순위 | 테스트 수 |
|------|------|---------|-----------|
| 인증 | 로그인, 회원가입, 소셜로그인, 로그아웃 | 🔴 Critical | 12 |
| 온보딩 | 신체정보, 식단선호, 음식취향 | 🟠 High | 8 |
| 재고관리 | CRUD, 영수증OCR, 유통기한 | 🔴 Critical | 15 |
| 레시피 | 목록, 추천, 맞춤형, 대체재료 | 🟠 High | 12 |
| 식사기록 | 촬영, AI분석, 피드백 | 🟠 High | 10 |
| 가족공유 | 초대, 참여, 권한 | 🟡 Medium | 8 |
| 커뮤니티 | 라운지, 쇼츠, 댓글 | 🟡 Medium | 10 |
| 알림 | 유통기한, 푸시 | 🟡 Medium | 5 |

**총 테스트 케이스: 80개**

---

## 테스트 케이스

### 1. 인증 (Auth)

#### TC-AUTH-001: 이메일 회원가입 성공
- **전제 조건**: 미가입 이메일
- **입력**: 이메일, 비밀번호, 이름
- **기대 결과**: 가입 성공, 온보딩 화면 이동
- **우선순위**: 🔴 Critical

#### TC-AUTH-002: 회원가입 실패 - 중복 이메일
- **전제 조건**: 이미 가입된 이메일
- **입력**: 중복 이메일
- **기대 결과**: "이미 가입된 이메일입니다" 에러
- **우선순위**: 🔴 Critical

#### TC-AUTH-003: 이메일 로그인 성공
- **전제 조건**: 가입된 사용자
- **입력**: 이메일, 비밀번호
- **기대 결과**: 로그인 성공, 홈 화면 이동
- **우선순위**: 🔴 Critical

#### TC-AUTH-004: 로그인 실패 - 잘못된 비밀번호
- **전제 조건**: 가입된 사용자
- **입력**: 이메일, 틀린 비밀번호
- **기대 결과**: "비밀번호가 일치하지 않습니다" 에러
- **우선순위**: 🔴 Critical

#### TC-AUTH-005: 카카오 로그인 성공
- **전제 조건**: 카카오 계정
- **입력**: 카카오 OAuth 토큰
- **기대 결과**: 로그인 성공
- **우선순위**: 🟠 High

#### TC-AUTH-006: 구글 로그인 성공
- **전제 조건**: 구글 계정
- **입력**: 구글 OAuth 토큰
- **기대 결과**: 로그인 성공
- **우선순위**: 🟠 High

#### TC-AUTH-007: 로그아웃
- **전제 조건**: 로그인 상태
- **입력**: 로그아웃 버튼 클릭
- **기대 결과**: 로그아웃, 로그인 화면 이동
- **우선순위**: 🟠 High

#### TC-AUTH-008: 토큰 만료 시 자동 갱신
- **전제 조건**: 만료된 access token, 유효한 refresh token
- **입력**: API 요청
- **기대 결과**: 토큰 자동 갱신, 요청 재시도
- **우선순위**: 🔴 Critical

### 2. 온보딩 (Onboarding)

#### TC-ONBOARD-001: 신체정보 입력
- **입력**: 키, 몸무게, 나이, 성별, 활동량
- **기대 결과**: 저장 후 다음 단계 이동
- **우선순위**: 🟠 High

#### TC-ONBOARD-002: 식단 선호도 선택
- **입력**: 당뇨식단, 저탄고지 등 선택
- **기대 결과**: 복수 선택 가능, 저장
- **우선순위**: 🟠 High

#### TC-ONBOARD-003: 음식 취향 선택 (넷플릭스 스타일)
- **입력**: 5개 이상 음식 선택
- **기대 결과**: 최소 5개 선택 시 완료 버튼 활성화
- **우선순위**: 🟠 High

#### TC-ONBOARD-004: 온보딩 건너뛰기
- **입력**: 건너뛰기 버튼
- **기대 결과**: 기본값으로 저장, 홈 이동
- **우선순위**: 🟡 Medium

### 3. 재고관리 (Inventory)

#### TC-INV-001: 재고 수동 추가
- **입력**: 품목명, 수량, 단위, 유통기한, 보관위치
- **기대 결과**: 재고 목록에 추가됨
- **우선순위**: 🔴 Critical

#### TC-INV-002: 영수증 OCR 스캔
- **입력**: 영수증 사진
- **기대 결과**: 품목 자동 인식, 확인 화면 표시
- **우선순위**: 🔴 Critical

#### TC-INV-003: OCR 결과 수정
- **전제 조건**: OCR 결과 화면
- **입력**: 품목명/수량 수정
- **기대 결과**: 수정된 값으로 저장
- **우선순위**: 🟠 High

#### TC-INV-004: 재고 수량 감소 (소비)
- **전제 조건**: 재고 존재
- **입력**: 소비 수량 입력
- **기대 결과**: 수량 감소, 0이면 삭제 확인
- **우선순위**: 🔴 Critical

#### TC-INV-005: 재고 삭제
- **전제 조건**: 재고 존재
- **입력**: 삭제 버튼
- **기대 결과**: 삭제 확인 후 목록에서 제거
- **우선순위**: 🟠 High

#### TC-INV-006: 유통기한 임박 표시
- **전제 조건**: D-3 이내 재고 존재
- **기대 결과**: 빨간색/주황색 배지 표시
- **우선순위**: 🟠 High

#### TC-INV-007: 카테고리 필터링
- **입력**: 카테고리 선택 (채소, 육류 등)
- **기대 결과**: 해당 카테고리만 표시
- **우선순위**: 🟡 Medium

#### TC-INV-008: 보관위치 필터링
- **입력**: 냉장/냉동/실온 선택
- **기대 결과**: 해당 위치만 표시
- **우선순위**: 🟡 Medium
