# 2. 디자인

## 호출
```
@design [요청사항]
```

## 역할
요구사항을 기반으로 **디자인 컨셉**을 정의하고, **프로토타입**을 생성합니다.

---

## 전제 조건

시작 전 확인:
- [ ] `docs/1-요구사항/main.md` 존재 여부

> ⚠️ 없으면: "요구사항 문서가 없습니다. `@requirements`로 먼저 요구사항을 정의해주세요."

---

## 입력
- `docs/1-요구사항/main.md` - 요구사항 문서
- `docs/1-요구사항/userflow.html` - 사용자 플로우
- (선택) 외부 디자인 이미지, 레퍼런스 URL

---

## 진행 순서

### Step 1: 디자인 컨셉 결정

사용자에게 질문:
1. **분위기**: 어떤 느낌을 원하시나요? (모던 / 미니멀 / 화려한 / 귀여운)
2. **주요 색상**: 선호하는 색상이 있나요?
3. **레퍼런스**: 참고하고 싶은 서비스나 이미지가 있나요?

> 💡 "모르겠다"면 → 제품 유형에 맞는 3가지 스타일 제안

### Step 2: 화면 구성

요구사항의 사용자 플로우를 기반으로:
1. 필요한 화면 목록 도출
2. 화면별 주요 컴포넌트 정의
3. 화면 간 네비게이션 플로우 작성

### Step 3: 프로토타입 생성

HTML/CSS로 클릭 가능한 프로토타입 생성

---

## 산출물

### 1. 디자인 컨셉: `docs/2-디자인/concept.md`

```markdown
# 디자인 컨셉

## 스타일
- **분위기**: [모던 / 미니멀 / etc]
- **톤앤매너**: [설명]

## 색상 팔레트
| 용도 | 색상 | HEX |
|------|------|-----|
| Primary | 🔵 | #3B82F6 |
| Secondary | ⚪ | #F3F4F6 |
| Accent | 🟢 | #10B981 |
| Text | ⚫ | #1F2937 |

## 타이포그래피
- **제목**: Pretendard Bold
- **본문**: Pretendard Regular
- **크기**: 16px 기준

## 레퍼런스
- [참고 서비스/이미지]
```

### 2. 화면 구성: `docs/2-디자인/screens.md`

```markdown
# 화면 구성

## 화면 목록
| ID | 화면명 | 설명 | 주요 컴포넌트 |
|----|--------|------|---------------|
| S-01 | 홈 | 메인 화면 | Header, Hero, FeatureList |
| S-02 | 로그인 | 로그인 화면 | LoginForm, SocialLogin |

## 네비게이션 플로우
S-01 홈 → S-02 로그인 → S-03 대시보드

## 화면별 상세

### S-01: 홈
- **목적**: [화면의 목적]
- **컴포넌트**:
  - Header: 로고, 네비게이션, 로그인 버튼
  - Hero: 메인 카피, CTA 버튼
  - FeatureList: 주요 기능 3개
```

### 3. 프로토타입: `docs/2-디자인/prototype.html`

HTML/CSS 기반 클릭 가능한 프로토타입
- 실제 화면 크기
- 네비게이션 동작
- 반응형 (선택)

---

## 디자인 원칙

### 1. 일관성
- 동일한 액션 = 동일한 스타일
- 컴포넌트 재사용

### 2. 계층 구조
- 중요한 것은 크고 대비있게
- 시선 흐름 고려 (Z-패턴, F-패턴)

### 3. 접근성
- 색상 대비 4.5:1 이상
- 터치 타겟 44px 이상
- 의미있는 텍스트 레이블
