# 1. 요구사항 정의

## 호출
```
@requirements [요청사항]
```

## 역할
사용자의 아이디어나 요구사항을 체계적으로 분석하고, **고객 관점**의 요구사항 문서로 구조화합니다.

---

## 동작 모드 (자동 판단)

시작 시 `docs/1-요구사항/main.md` 확인:

| 파일 상태 | 모드 | 동작 |
|-----------|------|------|
| 없음 | **신규 작성** | 질문으로 요구사항 수집 → 문서 생성 |
| 있음 | **수정/추가** | 기존 문서 요약 → "어떤 부분을 수정할까요?" |

---

## 명확화 질문

신규 모드에서 다음 질문으로 요구사항 수집:

1. **문제**: 어떤 문제를 해결하고 싶으신가요?
2. **해결방법**: 이 문제를 어떻게 해결하려 하시나요?
3. **성공 기준**: 성공적인 결과는 어떤 모습인가요?
4. **플랫폼**: 웹 / 모바일 / PC 중 어디서 사용하나요?
5. **사용 흐름**: 사용자가 어떤 순서로 사용하나요?

> 💡 사용자가 "모르겠다"고 하면, AI가 3가지 옵션을 제안하고 선택하게 합니다.

## 산출물

### 1. 메인 문서: `docs/1-요구사항/main.md`

```markdown
# [프로젝트명] 요구사항

## 문제 (Problem)
[해결하려는 문제 설명]

## 해결방법 (Solution)
[제안하는 해결 방법]

## 목표 (Goal)
[성공 기준, 측정 가능한 목표]

## 지원 플랫폼
| 플랫폼 | 지원 | 비고 |
|--------|------|------|
| 웹 | ✅/❌ | |
| iOS | ✅/❌ | |
| Android | ✅/❌ | |

## 사용자 플로우
1. [첫 번째 단계]
2. [두 번째 단계]
3. ...

## 요구사항 체크리스트
- [ ] REQ-001: [요구사항 1]
- [ ] REQ-002: [요구사항 2]
```
