# AI Agent Framework v2

AI 에이전트와 함께 웹/앱 서비스를 빠르게 만들기 위한 프레임워크

## 전체 흐름

```
1-요구사항 → 2-디자인 → 3-개발 → 4-테스팅 → 완료!
```

## 빠른 시작

| 단계 | 호출 | 설명 |
|------|------|------|
| 1 | `@requirements` | 아이디어 → 요구사항 문서화 |
| 2 | `@design` | 요구사항 → 디자인 컨셉 + 프로토타입 |
| 3a | `@frontend` | 디자인 → 프론트엔드 개발 |
| 3b | `@backend` | 기능 명세 → 백엔드 개발 |
| 4 | `@testing` | 통합 테스트 + QA |

## 디렉토리 구조

```
.agent-v2/
├── README.md                 # 👈 지금 보고 있는 파일

│   ├── 1-requirements/       # 요구사항 정의

│   ├── 3-development/        # 개발
│   │   ├── frontend/
│   │   └── backend/
│   └── 4-testing/            # 테스팅 + QA
│
├── knowledge/                # 📚 공유 지식
│   ├── ui-patterns/          # UI/UX 패턴
│   └── tech-stacks/          # 기술 스택 가이드
│
└── shared-rules/             # 📏 공통 규칙

```

## 산출물 구조

모든 산출물은 `docs/` 폴더에 저장됩니다:

```
docs/
├── 1-요구사항/
│   ├── main.md              # 메인 요구사항
│   └── userflow.html        # 사용자 플로우
├── 2-디자인/
│   ├── concept.md           # 디자인 컨셉
│   ├── screens.md           # 화면 구성
│   └── prototype.html       # 프로토타입 미리보기
├── 3-개발/
│   ├── frontend-spec.md     # 프론트엔드 명세
│   └── backend-spec.md      # 백엔드 명세
└── 4-테스팅/
    ├── test-plan.md         # 테스트 계획
    └── qa-report.md         # QA 리포트
```

## 사용 예시

### 처음 시작할 때
```
@requirements 쇼핑몰 만들어줘
```

### 중간부터 이어갈 때
```
@design 디자인 시작해줘
```
→ 이전 단계 산출물이 있으면 자동으로 읽어서 진행

### 특정 단계만 다시 할 때
```
@frontend 컴포넌트 구조 다시 설계해줘
```

## 핵심 원칙

1. **한 폴더에 다 있음**: 각 단계에 필요한 건 `phases/{단계}/AGENT.md`에 통합
2. **이전 단계 자동 참조**: 각 에이전트는 이전 산출물을 자동으로 읽음
3. **고객 관점 유지**: 기술 스펙이 아닌 "고객이 원하는 것" 중심
4. **한국어 응답**: 모든 응답은 한국어 (기술 용어는 영문 OK)

## 다음 단계로 자동 연결

각 단계 완료 시 다음 단계를 안내합니다:

```
✅ 요구사항 정의 완료!
→ 다음: @design 디자인 시작해줘
```
