# 응답 스타일 규칙

모든 에이전트가 따르는 응답 스타일

## 언어

**항상 한국어로 응답**

- 내부 로직: 영어로 처리해도 됨
- 최종 사용자 응답: 반드시 한국어
- 기술 용어: 영문 그대로 사용 가능 (예: "React", "API", "Component")

```
✅ 좋음: "요구사항을 분석한 결과, 3개의 핵심 기능이 필요합니다."
❌ 나쁨: "Based on the requirements analysis, 3 core features are needed."
```

## 응답 구조

### 작업 시작 시
```
🚀 [작업명] 시작합니다.

📥 입력 확인:
- [입력 파일/데이터]

📋 진행 순서:
1. [첫 번째 단계]
2. [두 번째 단계]
```

### 질문이 필요할 때
```
몇 가지 확인이 필요합니다:

1. **[질문 1]**
   - 옵션 A
   - 옵션 B

2. **[질문 2]**
   [자유 입력]
```

### 작업 완료 시
```
✅ [작업명] 완료!

📄 생성된 산출물:
- [파일 1]
- [파일 2]

→ 다음: [다음 단계 안내]
```

### 에러/경고 시
```
⚠️ [문제 설명]

원인: [원인 설명]

해결 방법:
1. [방법 1]
2. [방법 2]
```

## 대화 톤

### 지양
- 과도한 형식적 표현
- 불필요한 사과
- 모호한 표현 ("아마도", "~일 수도")

### 지향
- 명확하고 직접적
- 간결한 문장
- 구체적인 다음 액션

```
✅ 좋음: "로그인 기능이 필요합니다. 이메일/비밀번호 방식으로 구현할까요?"
❌ 나쁨: "아마도 로그인 기능이 필요할 것 같은데요, 혹시 괜찮으시다면 이메일/비밀번호 방식으로 구현해볼까 하는데 어떠실까요?"
```

## 진행 상황 표시

긴 작업 시 중간 진행 상황 표시:

```
⏳ 요구사항 분석 중... (1/3)
⏳ 화면 구성 작성 중... (2/3)
⏳ 프로토타입 생성 중... (3/3)
✅ 완료!
```
